# 发布与分发

本项目的用户脚本可以通过脚本平台（例如 GreasyFork）分发，也可以通过仓库 Raw 地址直接安装。

## 版本号

- 每次发布前更新脚本头部的 `@version`。
- 建议遵循语义化版本：修复误翻/漏翻为 patch，新增大批词条为 minor，重大机制变更为 major。

## 通过 GreasyFork 分发（以 old2.user.js 为例）

`scripts/old2.user.js` 包含 GreasyFork 常见的发布字段：

- `@downloadURL`：用户脚本本体下载地址
- `@updateURL`：元数据更新地址
- `@require`：外部词库脚本地址（用于词库与逻辑分离）

发布建议：

1. 确认 `@match` 覆盖范围符合预期，避免误伤非目标站点。
2. 若使用 `@require`，优先使用稳定的更新域名，并在文档中明确“词库版本与脚本版本”的对应关系。
   - 词库文件建议以“普通脚本”形式写入 `window.VERCEL_I18N_DATA`，避免模块语法。
3. 发布后用全新浏览器配置验证“安装 → 自动更新 → 生效”链路。

## 直接通过仓库分发

如果通过仓库分发，建议在 README 与 RepoWiki 中提供一个明确的安装入口（指向具体脚本文件），并保持脚本头部元数据（名称、版本、匹配范围）与仓库内容同步。
