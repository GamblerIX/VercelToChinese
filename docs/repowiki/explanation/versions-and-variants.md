# 脚本变体与取舍

仓库内目前同时保留了两份历史脚本（`old1.user.js` 与 `old2.user.js`）。它们的目标一致，但在“词库来源、性能策略、分发方式”上取舍不同。

## old1.user.js

- 词库：内置在脚本内（`I18N` 对象）
- 规则：少量正则规则（`REGEX_RULES`）
- 引擎：队列 + 空闲切片 + 缓存；对大页面更友好
- 覆盖：包含 `*.vercel.app/*`，范围更大

适合场景：

- 不希望依赖外部词库加载
- 更关注性能与稳定性（减少递归遍历导致的长任务）

## old2.user.js

- 词库：通过 `@require` 外部加载（`window.VERCEL_I18N_DATA`）
- 引擎：逻辑更简单，递归遍历为主
- 覆盖：不包含 `vercel.app`，范围更保守

适合场景：

- 希望词库与逻辑分离、词库可独立更新
- 需要对外发布并保持脚本主体较轻

## 建议的收敛方向

- 明确“默认推荐使用的脚本变体”，并把另一份标记为历史实现（仅作参考）。
- 统一词库维护入口：
  - 要么内置（单文件更易安装）
  - 要么外置（便于协作维护与独立发布）
